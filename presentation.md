# 時間割Generator 発表準備メモ

## 1. アプリ概要

### アプリ名

時間割Generator

### 目的

大学の履修科目を管理し、時間割を視覚的に作成するためのWebアプリ。

従来は時間割PDFから科目情報を人手で確認し、履修候補を手作業で整理する必要があった。  
このアプリでは、PDFから科目マスタを取り込み、フィルタリング・検索・履修登録・履修一覧確認までを1つの画面で行える。

### 使用技術

- バックエンド: FastAPI
- テンプレート: Jinja2
- フロントエンド更新: HTMX
- データベース: SQLite
- ORM: SQLModel
- PDF解析: pdfplumber
- スタイル: Tailwind CSS CDN + 独自CSS

### 基本的な利用の流れ

1. 時間割PDFを取り込みページからアップロードする。
2. PDF解析結果をプレビューで確認する。
3. 確定すると科目マスタがデータベースに登録される。
4. メイン画面で回生・分野・登録区分を指定して科目を絞り込む。
5. 時間割の空きコマをクリックして、右サイドバーから科目を履修登録する。
6. 履修一覧サイドバーで登録済み科目と集計を確認する。
7. 必要に応じて履修解除や再登録を行う。

## 2. 現状の主要機能

### 2.1 時間割表の表示

メイン画面では、月曜から金曜、1限から5限までの時間割グリッドを表示する。

表示される情報:

- 曜日: 月曜、火曜、水曜、木曜、金曜
- 時限: 1限から5限
- 時間帯:
  - 1限: 8:45-10:15
  - 2限: 10:30-12:00
  - 3限: 13:00-14:30
  - 4限: 14:45-16:15
  - 5限: 16:30-18:00

履修登録済みのコマには、科目カードが表示される。  
科目カードには科目名、科目分野、解除ボタンが表示され、分野ごとに背景色が変わる。

### 2.2 履修登録機能

空いている時間割セルをクリックすると、右サイドバーにその曜日・時限で履修可能な科目一覧が表示される。

登録の流れ:

1. 空きコマをクリックする。
2. 右サイドバーに該当コマの科目一覧が表示される。
3. 科目の「登録」ボタンを押す。
4. 該当セルに科目カードが追加される。
5. サイドバーは履修一覧に戻る。

登録時の制約:

- 同じ曜日・同じ時限には1科目のみ登録可能。
- すでに登録済みの科目を重複登録することはできない。
- 存在しない科目IDは登録できない。

実装上のポイント:

- `POST /api/enrollment` で履修登録する。
- `crud.add_enrollment()` で重複チェックを行う。
- 登録成功時は `partials/enrolled_cell.html` を返し、HTMXで対象セルだけを更新する。
- 画面全体をリロードせずに操作できる。

### 2.3 履修解除機能

登録済み科目カードの「解除」ボタンから履修解除できる。

解除できる場所:

- 時間割セル内の科目カード
- 右サイドバーの履修一覧

解除後の動作:

- 該当セルが空きコマ表示に戻る。
- 履修一覧から解除した場合は、一覧とサマリーが再読み込みされる。

実装上のポイント:

- `DELETE /api/enrollment/{enrollment_id}` を使用する。
- 削除成功時は空セル用HTMLを返す。
- HTMXで該当セルだけを更新する。

### 2.4 フィルタリング機能

メイン画面上部のフィルタで、表示・検索対象の科目を絞り込める。

フィルタ項目:

- 回生
  - 1回生、2回生、3回生、4回生
  - 排他選択
- 科目分野
  - 外国語
  - 基礎専門
  - 固有専門
  - 共通専門
  - 教養
  - 全て
  - 複数選択可能
- 登録区分
  - 事務室登録
  - 抽選
  - その他
  - 全て
  - 複数選択可能

仕様:

- 回生は `Subject.year <= 選択回生` で絞り込む。
- 科目分野は複数選択時に `IN` 条件で絞り込む。
- 登録区分も複数選択に対応する。
- 「その他」は「事務室登録」「抽選」以外を対象にする。
- フィルタ変更時、科目選択サイドバーが開いていれば自動で再取得する。

実装上のポイント:

- フロント側では選択中の値をカンマ区切りで保持する。
- バックエンド側の `parse_multi_value()` でリストに変換する。
- `crud._apply_subject_filters()` が共通の絞り込み処理を担当する。

### 2.5 科目検索機能

科目名または授業コードで科目を検索できる。

検索の特徴:

- 科目名の部分一致検索に対応。
- 授業コードの部分一致検索に対応。
- 回生・科目分野・登録区分フィルタと併用できる。
- 検索結果から直接履修登録できる。
- 登録後は履修一覧サイドバーに戻る。

実装上のポイント:

- `GET /api/subjects/search` を使用する。
- `crud.search_subjects()` で科目名・授業コードを検索する。
- 結果は `partials/search_results.html` として右サイドバーに差し込まれる。

### 2.6 履修一覧・サマリー表示

起動時には右サイドバーに履修一覧が表示される。

表示内容:

- 履修登録数
- 曜日別の登録数
- 分野別の登録数
- 登録区分別の登録数
- 登録済み科目一覧

履修一覧の役割:

- 現在の履修状況を一覧で確認できる。
- 時間割表だけでは見落としやすい登録済み科目を俯瞰できる。
- 一覧から履修解除できる。
- 起動時・登録後・解除後に常にサイドバーとして表示できる。

実装上のポイント:

- `GET /api/enrollment/summary` を使用する。
- `build_enrollment_summary()` で曜日別・分野別・登録区分別に集計する。
- 初期表示時にもサマリー用データをテンプレートへ渡している。
- サイドバーを閉じる操作をしても、完全に非表示にはせず履修一覧へ戻す設計にしている。

### 2.7 常時サイドバー表示

アプリ起動時には右側に履修一覧が表示される。

サイドバーの状態:

- 起動時: 履修一覧
- 時間割セルクリック時: 科目選択
- 科目検索時: 検索結果
- 履修登録後: 履修一覧
- 履修一覧の解除後: 履修一覧
- 右上の閉じるボタン押下時: 履修一覧

この設計により、画面右側が空になる場面を減らし、常に履修状況または次の操作候補を表示できる。

### 2.8 PDF取り込み機能

時間割PDFをアップロードし、科目マスタを作成できる。

取り込みの流れ:

1. `/import` にアクセスする。
2. 時間割PDFをアップロードする。
3. ページ指定方法を選ぶ。
4. プレビューを表示する。
5. 解析結果を確認する。
6. 問題なければ取り込みを確定する。

ページ指定方式:

- 開始ページから連番
  - 月曜のページを指定すると、火曜から金曜は続くページとして扱う。
- 曜日ごとに指定
  - 月曜から金曜まで個別にページ番号を指定する。

バリデーション:

- PDFファイルのみ受付。
- ファイルサイズ上限は20MB。
- ページ番号は正の整数。
- 指定ページがPDFの範囲外なら警告を表示する。

実装上のポイント:

- `POST /api/import/preview` でPDFを解析し、DBは更新しない。
- `app/pdf_parser.py` がPDF表データを解析する。
- `app/import_store.py` がプレビュー結果を一時保存する。
- `POST /api/import/confirm` でプレビュー済みデータをDBへ反映する。
- 確定時は既存の科目マスタを全置換し、履修登録もすべて解除する。

### 2.9 PDF取り込みプレビュー

PDFをアップロードしても、すぐにDBは更新されない。  
まずプレビュー画面で抽出結果を確認できる。

プレビューで確認できる内容:

- 抽出された科目数
- 曜日別件数
- 警告一覧
- 科目一覧
  - 曜日
  - 時限
  - 授業コード
  - 科目名
  - 分野
  - 回生
  - 登録区分

メリット:

- 誤ったPDFやページ指定ミスを確定前に発見できる。
- PDF解析がレイアウト依存であるため、プレビュー確認により安全性を高められる。
- 取り込み確定前に、既存データが置き換わることを明示できる。

### 2.10 ライトモード・ダークモード切り替え

画面上部のテーマ切替ボタンから、ライトモードとダークモードを切り替えられる。

特徴:

- デフォルトはダークモード。
- ライトモード選択時は、メイン画面だけでなく科目データ取り込みページにも適用される。
- 選択状態は `localStorage` に保存される。
- 次回アクセス時も前回選択したテーマが復元される。
- CSSキャッシュ対策として、`style.css` の読み込みURLにバージョン文字列を付けている。

実装上のポイント:

- `body.theme-light` によってライトテーマ用CSSを適用する。
- ページ読み込み前に `theme-light-preload` を付与し、表示直後のちらつきを抑える。
- メイン画面では `toggleTheme()` によりテーマを切り替える。
- 取り込みページでは保存済みテーマを読み取り、同じ見た目を適用する。

### 2.11 UIデザイン

基本方針:

- ダークテーマを基調とした時間割管理画面。
- グラス風のカード表現。
- 科目分野ごとの色分け。
- 右サイドバーで文脈に応じた情報を表示。
- HTMXにより画面全体をリロードせず、必要な部分のみ更新する。

主なUI要素:

- 時間割グリッド
- フィルタボタン
- 科目検索入力
- 履修一覧サイドバー
- PDF取り込みフォーム
- プレビュー一覧
- ライト・ダーク切替ボタン

## 3. データモデル

### Subject

科目マスタを表す。

主な項目:

- `id`: 主キー
- `code`: 授業コード
- `reg`: 登録区分
- `name`: 科目名
- `cat`: 科目分野
- `year`: 回生
- `day`: 曜日
- `period`: 時限

### Enrollment

履修登録を表す。

主な項目:

- `id`: 主キー
- `subject_id`: `Subject` への外部キー

履修登録テーブルは科目IDだけを持ち、表示時に `Subject` とJOINして科目名・曜日・時限などを取得する。

## 4. 主なAPI

| エンドポイント | メソッド | 役割 |
|---|---|---|
| `/` | GET | メインの時間割画面を表示 |
| `/import` | GET | PDF取り込み画面を表示 |
| `/api/subjects/{day}/{period}` | GET | 指定コマの科目一覧を返す |
| `/api/subjects/search` | GET | 科目名・授業コード検索結果を返す |
| `/api/enrollment` | GET | 履修登録一覧をJSONで返す |
| `/api/enrollment/summary` | GET | 履修一覧・サマリーをHTMLで返す |
| `/api/enrollment` | POST | 履修登録する |
| `/api/enrollment/{enrollment_id}` | DELETE | 履修解除する |
| `/api/import/preview` | POST | PDFを解析してプレビューを返す |
| `/api/import/confirm` | POST | プレビュー済みデータをDBへ反映する |
| `/api/import/cancel` | POST | 取り込みプレビューを破棄する |

## 5. 技術的な特徴

### HTMXによる部分更新

画面全体のリロードではなく、必要なHTML断片だけを返して差し替える。

例:

- 科目一覧サイドバーの表示
- 検索結果の表示
- 履修登録後のセル更新
- 履修解除後のセル更新
- 履修一覧サマリーの再表示
- PDF取り込みプレビューの表示

### PDF解析とプレビューの分離

PDF解析後すぐにDBへ保存せず、一時ファイルにプレビュー結果を保存する。

利点:

- ユーザーが内容を確認してから確定できる。
- ページ指定ミスやPDF形式不一致を検出しやすい。
- 確定前ならキャンセルできる。

### シンプルなDB設計

科目マスタ `Subject` と履修登録 `Enrollment` の2テーブル構成。  
履修登録は `subject_id` の参照だけを保持し、科目情報はマスタから取得する。

### テーマ状態の永続化

ライト・ダークモードの選択を `localStorage` に保存することで、サーバー側のDBやログイン機能なしでユーザー設定を維持する。

- フィルタリングと検索により、大量の科目から目的の科目を探しやすい。
- 履修登録と解除が直感的で、時間割表にすぐ反映される。
- 履修一覧サイドバーで、登録済み科目と集計を常に確認できる。
- ライト・ダークモードに対応し、見た目の好みに合わせられる。
- FastAPI + HTMX により、シンプルな構成でインタラクティブなUIを実現している。

## 7. 現状の制約・今後の改善案

### 現状の制約

- PDFパーサは特定の表レイアウトに依存している。
- 科目マスタ確定時には、既存科目と履修登録が全て置き換わる。
- 単位数や必修・選択必修などの卒業要件チェックには未対応。
- 複数の時間割案を保存する機能は未実装。
- ログイン機能はなく、ローカル利用を前提としている。

### 今後の改善案

- 履修データのエクスポート・インポート
- 複数時間割案の保存
- 単位数管理
- PDF取り込みプレビュー上での手動修正
- 印刷用・共有用の時間割出力
- 大学や年度ごとのPDFレイアウト設定
